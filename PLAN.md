# 中国象棋完整版实施计划（EasyX 已安装 / 11x10 特殊模式修订版）

## 摘要
- 目标是把当前的 Visual Studio C++ 工程扩展为完整的中国象棋项目，正式范围包含文档任务 1-4，并把人机对战纳入必做项。
- 交付形态固定为“单规则内核 + 多入口前端”：同一套规则引擎同时服务控制台模式、EasyX 图形界面、局域网主机权威对战、棋谱回放、智能提示和人机对战。
- EasyX 视为本机现有依赖，实施时只做工程对接，不安排重新安装。
- 局域网功能支持主机权威对战、房间发现、观战和黑方断线重连。
- 游戏模式固定为两种：标准 `9x10` 和特殊模式 `11x10`。特殊模式保持所有棋子走法不变，只扩展棋盘宽度。

## 核心接口与类型
- `enum class BoardMode { Standard9x10, Expanded11x10 };`
- `enum class Side { Red, Black };`
- `enum class PieceType { King, Advisor, Elephant, Knight, Rook, Cannon, Pawn };`
- `enum class GameResult { Ongoing, RedWin, BlackWin, Draw, Timeout, Resign };`
- `template<typename T> struct Position { T row; T col; };` 作为统一坐标模板，重载 `==` 和 `!=`。
- `struct BoardConfig` 固定包含 `mode`、`rows`、`cols`、`river_split`、九宫边界、初始摆子表、坐标字母表。
- `struct Move` 固定包含起点、终点、执行方、移动棋子类型、被吃棋子类型、被吃前信息、走后是否将军、走后是否将死。
- `struct ParsedCommand` 固定包含命令类型、原始输入、可选走法、可选参数字符串。
- `class ChessPiece` 为抽象基类，公开 `type()`、`side()`、`symbol()`、`value()`、`clone()`、`isPseudoLegal(...)`、`generatePseudoLegalMoves(...)`，并基于 `value()` 重载 `<` 和 `>`。
- 七个派生类分别实现将、士、象、马、车、炮、兵的伪合法走法逻辑。
- `class Board` 使用一维动态容器保存 `rows * cols` 个格点，元素为 `std::unique_ptr<ChessPiece>`，通过 `index(row, col)` 映射，适配 9 列和 11 列两种模式。
- `Board` 公开 `setup(BoardConfig)`、`pieceAt()`、`isInside()`、`countObstacles()`、`isPalaceCell()`、`hasFlyingKings()`、`isSquareAttacked()`、`generatePseudoLegalMoves()`、`generateLegalMoves()`、`applyMove()`、`revertMove()`。
- `Board` 重载 `operator+(const Move&)`、`operator-(const Move&)`、`operator==`、`ostream << Board`。
- `class MoveParser` 统一负责坐标输入、标准棋盘中文记谱输入和控制命令输入，重载 `istream >> ParsedCommand`。
- `class GameSession` 作为唯一状态入口，持有 `Board`、当前行棋方、历史栈、重复局面哈希、双方玩家、计时、设置、结果和当前模式。
- `class ReplayRecorder` 与 `class ReplayPlayer` 负责棋谱记录和回放。
- `class NetworkSession` 负责局域网主机/客户端通信，主机为权威状态源。
- `class SearchEngine` 负责提示和人机对战，统一使用 alpha-beta 搜索。
- 异常类型固定为 `InputError`、`IllegalMoveError`、`StorageError`、`NetworkError`、`ResourceError`。

## 规则与模式规格
- 标准模式固定为 `9 列 x 10 行`，坐标范围为列 `a-i`、行 `0-9`。
- 特殊模式固定为 `11 列 x 10 行`，坐标范围为列 `a-k`、行 `0-9`。
- 两种模式都保留河界在 `row 4` 和 `row 5` 之间，红黑前进方向不变。
- 两种模式都保留九宫为 `3x3`，上方九宫为 `row 0-2`，下方九宫为 `row 7-9`。
- 标准模式九宫列范围固定为 `col 3-5`。
- 特殊模式九宫列范围固定为 `col 4-6`，即在 11 列棋盘中保持居中。
- 特殊模式初始摆法采用“标准 9 列布局整体右移 1 列”的方案，左右两侧各多出 1 条空列。
- 特殊模式顶底线摆子固定为：车在 `1/9`，马在 `2/8`，象在 `3/7`，士在 `4/6`，将帅在 `5`。
- 特殊模式炮固定在 `2/8` 列，兵卒固定在 `1/3/5/7/9` 列。
- 特殊模式中，象不过河、士和将仍受九宫限制、兵卒前进方向不变、飞将规则不变，额外空列只是提供更宽活动空间。
- 标准模式支持坐标输入和中文记谱输入。
- 特殊模式不支持中文记谱输入，只支持坐标输入和 GUI 鼠标操作；回放文本也使用坐标记谱，避免 11 列文件位的中文编号歧义。
- 胜负规则固定为将死判负、困毙判负、认输判负、超时判负、三次重复局面判和。
- 长将和长捉通过重复局面、连续将军和循环追吃检测覆盖。

## 实施阶段
- 第一阶段重构工程结构，拆分当前单文件工程，建立 `engine`、`app`、`ui_console`、`ui_easyx`、`net`、`ai`、`storage`、`tests` 模块边界，并保持现有 `.sln` 与 `.vcxproj` 延续使用。
- 第一阶段同时完成 EasyX 对接，只做头文件目录、库目录、链接项和图形入口封装接入，不改用其他图形框架。
- 第二阶段实现 `BoardConfig`、`Position<T>`、`Move`、`ChessPiece` 体系和两种模式下的初始布局生成。
- 第二阶段完成七类棋子的伪合法走法、障碍判定、飞将、自将暴露、绝对合法走法筛选、将军、将死、困毙、重复局面判和。
- 第三阶段实现 `GameSession`，统一接管回合切换、历史栈、悔棋、计时、结果状态、模式选择、局面克隆和错误处理。
- 第三阶段完成运算符重载落点：`Position` 的 `==`/`!=`，`ChessPiece` 的 `<`/`>`，`Board` 的 `+`/`-`/`==`，`Board` 的 `<<`，命令解析的 `>>`。
- 第四阶段交付控制台完整版，支持字符棋盘、红黑颜色区分、合法落点提示、将军提示、坐标输入、标准模式中文记谱输入、`undo`、`save`、`load`、`resign`、`hint`、`replay`、`help`、`exit`。
- 第五阶段实现存档、排行榜、设置和计时。存档格式固定为文本 `.xqsave`，棋谱格式固定为标准化 `.pgn`，PGN 固定保存到 `D:\visualstudio\中国象棋\pgn` 并使用时间戳命名，排行榜固定为 `leaderboard.csv`。
- 第五阶段的设置项固定包括棋盘模式、每步限时、是否允许悔棋、是否显示合法走法提示、是否启用 AI、AI 搜索深度、默认启动界面。
- 第六阶段实现 EasyX 图形界面，支持菜单页、模式选择、棋盘绘制、鼠标选子与落子、合法落点高亮、最近一步高亮、当前方提示、计时显示、将军提醒、胜负弹窗、回放控制。
- 第六阶段 GUI 必须支持标准模式和 11x10 特殊模式，绘制逻辑按 `BoardConfig.cols` 动态适配，不能写死为 9 列。
- 第七阶段实现局域网主机权威模式，采用 WinSock TCP，主机负责建局、保存权威状态、验证命令、广播标准化结果，客户端只发送命令和接收同步。
- 第七阶段网络握手固定同步以下参数：棋盘模式、时间设置、是否允许悔棋、双方昵称、先手方。
- 第七阶段实现房间发现、只读观战和黑方断线重连；不实现跨公网匹配大厅和账号系统。
- 第八阶段实现棋谱记录与回放，支持逐步前进、逐步后退、自动播放、暂停、跳转指定回合；标准模式回放显示中文记谱，11x10 模式回放显示坐标记谱。
- 第九阶段实现智能提示与人机对战，提示与电脑落子共享 `SearchEngine`，搜索算法固定为 alpha-beta 极大极小搜索，难度固定为 2 层、3 层、4 层。
- 第九阶段评估函数固定包含子力值、将帅安全、兵卒过河奖励、机动性、将军压力和被攻击惩罚；AI 必须支持标准模式和 11x10 特殊模式。
- 第十阶段补齐课程交付材料，包括需求分析、类图、流程图、原型图、测试记录、演示脚本和代码说明，确保课程要求与代码实现一一对应。

## 关键实现约束
- 规则引擎必须是唯一真相来源，控制台、EasyX、联网、回放、AI 都只调用 `GameSession` 和 `Board` 公共接口，不得复制规则逻辑。
- 棋盘与棋子所有权统一使用 RAII 管理，不使用裸指针表达资源所有权。
- `Board` 和 `GameSession` 必须显式支持深拷贝，因为 AI 搜索、回放重建、悔棋和存档恢复都需要复制局面。
- `operator+` 和 `operator-` 必须返回新状态而不是原地修改，以便复用在搜索和回放中。
- 控制台和 GUI 均保留，二者属于同一可执行程序的不同入口模式，不拆成两个独立项目。
- 11x10 模式作为正式特殊模式参与菜单选择、存档、回放、排行榜和局域网握手，不能只做本地演示分支。
- 中文记谱解析仅对标准模式启用，11x10 模式统一退回坐标体系，避免额外创造非标准中文列编号。
- 存档必须保存完整模式信息，加载时可无损恢复标准模式或 11x10 模式的局面、设置和历史。
- 网络通信采用简洁文本协议即可，不引入额外网络库。

## 测试计划
- 单元测试覆盖两种棋盘模式下七类棋子的合法与非法走法、阻挡、边界、飞将、将军、自将暴露、将死、困毙和重复局面判和。
- 特殊模式测试必须覆盖 11 列棋盘的边缘移动、九宫位置、扩展空列利用、标准布局右移后初始落点是否正确。
- 输入测试必须覆盖标准模式坐标输入、标准模式中文记谱输入、11x10 模式坐标输入、11x10 模式拒绝中文记谱、非法命令、越界坐标和空输入。
- 控制台测试必须覆盖模式切换、颜色显示、合法落点提示、悔棋、存档、读档、回放、认输和超时。
- EasyX 测试必须覆盖标准模式和 11x10 模式的棋盘绘制、选中/取消、移动、提示高亮、最近一步高亮、胜负弹窗和回放操作。
- 联机测试必须覆盖主机建局、客户端连接、参数同步、正常走子、非法走子拒绝、悔棋请求与确认、超时、认输和结束同步。
- AI 测试必须保证在两种棋盘模式下只生成合法走法，并能在被将时优先寻找脱将方案。
- 存档与回放测试必须保证标准模式和 11x10 模式均可跨进程保存、加载、恢复和完整重放。
- 最终验收标准固定为：控制台和 GUI 均可完整下一局；局域网主机和客户端状态全程一致，并支持房间发现、观战和重连；标准模式与 11x10 模式都可存档、回放、计时和排行；AI 可在两种模式下完成对局；课程要求中的模板、继承、多态、运算符重载和异常处理都有清晰代码落点与演示案例。

## 假设与默认值
- 开发平台固定为 Windows、Visual Studio 2022、MSVC v143。
- EasyX 已存在于本机环境中，实施只做工程对接，不重装、不替换图形库。
- 局域网功能面向局域网直连，支持房间发现、观战和黑方断线重连。
- 特殊模式固定解释为“11 列 x 10 行”，不是 11 行 x 10 列。
- 特殊模式沿用标准规则，只扩大棋盘宽度并采用居中的标准初始布局。
- 标准模式支持中文记谱，11x10 模式不支持中文记谱。
- 重复局面、长将禁着和循环追吃检测共同处理长将/长捉裁定。
- 持久化格式使用轻量文本文件，不引入额外第三方序列化库。
